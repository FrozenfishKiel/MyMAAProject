# 项目介绍：基于深度学习和计算机视觉的游戏自动化脚本设计与实现

我们的项目分成了四个模块开发：

设备模块->视觉识别模块->决策模块->用户界面模块

主要分工如下：
设备模块主要是负责连接MUMU模拟器，通过ADB协议与模拟器进行通信，同时直接调用MUMU模拟器的SDK接口实现高性能屏幕捕获。该模块包含ADB控制器、设备管理器、输入控制器和屏幕捕获等子模块，实现了设备状态监控、连接管理、错误处理和重试机制。具体功能包括：通过ADB命令执行设备控制，使用MUMU模拟器的external_renderer_ipc.dll库绕过窗口系统直接获取渲染缓冲区，实现高性能的屏幕截图和输入操作（如点击、滑动、按键等）。设备模块还提供了统一的设备管理接口，支持多设备管理和配置切换，确保系统的稳定性和可靠性。这个模块的开发进度已经基本完成

视觉识别模块主要是负责从设备模块获取到的图像去进行预处理并识别出游戏中的元素，例如角色，道具，敌人等。我们使用了深度学习和计算机视觉技术，技术栈包括YOLOV8，OpenCV模板匹配，以及一些其他的计算机视觉库，这里识别的部分都是来自MaaAssistantArknights-dev/MaaCore/Vision的，因为如果要自己去识别的话需要重新收集游戏中的元素的截图，而这些截图已经被MaaAssistantArknights-dev/MaaCore/Vision的开发者收集并标注好了，所以我们直接使用他们的代码即可。这个模块的开发进度已经基本完成

决策模块主要是将视觉识别模块识别出的元素进行分析和判断，然后输入到PPO算法中，PPO算法是一种基于策略梯度的强化学习算法，它的目标是通过最大化奖励函数来学习最优策略。在我们的项目中，因为目前PPO的奖励机制和动作空间还在设计中且进度也到这了，而且这一部分就是项目的核心功能，也是最难的部分（我个人认为）。这个模块的开发进度尚未完成，没有参考价值，因为写的很乱很散，而且没有确定明确的奖励机制和动作空间等其他功能，所以现在就是卡在这个核心功能开发上

最后是用户界面模块，主要是负责与用户进行交互，例如显示演示画面，接收用户输入的指令等，这一模块因为还未开发，且不是项目核心功能，目前进度还未到，所以我当前的项目没有写到这个部分。

**设备模块文件位置**：`src/device/`
- ADB控制器：`src/device/adb_controller.py`
- 设备管理器：`src/device/device_manager.py`
- 输入控制器：`src/device/input_controller.py`
- 屏幕捕获：`src/device/screen_capture.py`
- 高性能捕获：`src/device/high_performance_capture/`
- 配置文件：`src/device/config.py`

**视觉识别模块文件位置**：`src/vision/`
- YOLO检测器：`src/vision/detector/yolo_detector.py`
- 图像处理器：`src/vision/processor/image_processor.py`
- 数据集：`data/datasets/`目录
- 训练配置：`config/yolo_training_config.yaml`

**决策模块文件位置**：`src/ppo/`
- 游戏环境：`src/ppo/game_environment.py`
- 状态编码器：`src/ppo/state_encoder.py`
- 动作空间：`src/ppo/action_space.py`

**用户界面模块文件位置**：`src/ui/`（计划开发）
