# 训练手册（YOLO / RL）

本手册面向“AI 增强模块”，目标是补齐复杂场景识别与自主决策。现阶段仓库提供接口与落盘结构，训练产物建议统一放入 `models/`，并在运行配置中引用。

## 1. 数据与采集

### 1.1 采集原则

- 覆盖分辨率/画质/光照/特效/语言等变量
- UI 元素需覆盖：出现/消失、遮挡、禁用态、动画态

### 1.2 数据落盘约定

- 训练图像：`data/dataset/images/`
- 标注文件：`data/dataset/labels/`
- Maa 模板：`data/templates/`

## 2. YOLO 训练（复杂目标检测）

### 2.1 适用场景

- Maa 模板匹配对动态/复杂光影/强动画场景稳定性不足时
- 需要检测多个目标（按钮/怪物/道具）并输出坐标

### 2.2 标注

- 使用 LabelImg/Roboflow 等工具生成 YOLO 格式标注
- 标签体系建议按“可点击 UI / 资源点 / 敌人 / 文字区域”分组

### 2.3 训练命令（示例）

以 Ultralytics YOLO 为例（需自行安装训练依赖）：

```bash
pip install ultralytics
yolo detect train data=path/to/data.yaml model=yolov8n.pt imgsz=640 epochs=100
```

### 2.4 评估与门槛

- 关注 mAP50/mAP50-95、误检率、漏检率
- 按“关键用例”的最小可用标准：关键按钮漏检率应尽量接近 0

### 2.5 导出与部署

- 权重放入：`models/yolo/best.pt`
- 运行时调用：`src/ai-plugins/yolo_recognizer.py` 的 `load()` + `detect()`

## 3. 强化学习训练（自主决策）

### 3.1 适用场景

- 探索性测试：地图探索、战斗策略、随机事件
- 固定任务链难以覆盖的策略空间

### 3.2 环境建模建议

- 状态（observation）：
  - 低维：血量/能量/关卡进度/关键 UI 是否出现（由识别器输出）
  - 高维：截图特征（不建议直接用原图起步）
- 动作（action）：可执行的“原子动作”集合（点击/滑动/等待/技能键）
- 奖励（reward）：
  - 完成任务 +1
  - 死亡/失败 -1
  - 时间/资源消耗惩罚（避免无意义循环）

### 3.3 训练工具链（示例）

以 Stable-Baselines3 为例（需自行安装训练依赖）：

```bash
pip install stable-baselines3 gymnasium
```

训练完成后保存：`models/rl/policy.zip`

运行时加载接口：`src/ai-plugins/rl_policy.py` 的 `load()` + `act()`。

### 3.4 线上化原则

- 先“规则兜底”：任何 RL 动作必须能回退到固定任务链
- 记录轨迹：关键状态、动作序列、失败截图，便于复现与迭代

## 4. 联动 Maa 的推荐策略

### 4.1 识别优先级

1) Maa 模板/特征匹配：快、稳定、适合固定 UI
2) YOLO：复杂场景兜底
3) OCR：用于错误弹窗/提示文本解析

### 4.2 决策优先级

1) 固定任务链：流程性强的场景
2) RL 决策：探索性强的节点
3) 安全回退：超时/连续失败则回到固定链或退出用例
