# MaaFramework 学习分析

## 架构分层

设备层 → 采集层 → 识别层 → 任务层 → 动作层

## 关键模块职责边界

- 设备管理：负责设备连接、状态监控、多设备并发调度入口
- 屏幕采集：高帧率、低延迟的帧获取与区域裁剪
- 图像识别：模板匹配/特征点/OCR 的基础识别能力
- 任务编排：任务链、条件分支、循环控制、失败恢复
- 动作执行：点击/滑动/输入/按键的跨平台适配

## Python 侧集成策略

- 100% 复用：设备管理、采集、基础识别、动作
- 通过扩展补齐：YOLO 识别器、RL 决策器、报告生成

## API 调用流程（本仓库实现）

`src/main.py` 加载配置 → 初始化运行时 → 执行任务链 → 写入结构化日志 → 生成 HTML/Excel 报告
